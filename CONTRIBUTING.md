# Contributing to EZPassRI


Thank you for your interest in contributing to **EZPassRI**! Your help makes this project better for everyone.


## Contribution Guidelines


Please follow these guidelines to keep the project healthy and maintainable:


- Write clear, descriptive commit messages.
- Keep pull requests small and focused.
- Write tests for new functionality where applicable.
- Follow existing code style and patterns.


## How to Contribute


1. Fork the repository.
2. Create a new branch named `feature/your-feature` or `fix/your-fix`.
3. Make your changes on the branch.
4. Run the test suite and linters (if any).
5. Commit and push your branch to your fork.
6. Open a Pull Request against `main` with a clear description of changes.


## Reporting Issues


If you find a bug or problem:


- Search existing issues first to avoid duplicates.
- Create an issue with a clear title and steps to reproduce.
- Include expected vs actual behavior and system details if relevant.


## Suggesting Features


- Open an issue labeled `enhancement` or `feature`.
- Describe the use-case and benefit.
- If possible, suggest a rough implementation plan.


## Code of Conduct


This project follows a [Contributor Covenant]-style code of conduct.


- Be respectful and inclusive.
- No harassment or hateful speech.
- Report abusive behavior to the maintainers.


## Review Process


- PRs will be reviewed by maintainers or assigned reviewers.
- Reviewers may request changes; please respond respectfully.
- After approval, a maintainer will merge the PR.
